ApiDelta — May 10, 2026

Mistral added a new model endpoint, Claude services recovered from several brief outages, and OpenAI reported resolved performance hiccups for Codex and the Responses API.

Mistral Small 3.2 model and endpoint now live

Mistral released Mistral Small 3.2 and opened the mistral-small-2506 API endpoint. Existing integrations can start sending requests to the new model version; no code changes are required other than updating the model identifier. Source

Claude IDE extensions and web UI experienced short‑term failures, now fixed

Windows Claude Code IDE extension, the Claude Code web UI, and several Claude model versions (Opus 4.1, Opus 4.7, Sonnet 4.6) suffered loading errors and elevated response failures on May 8‑9. All incidents are marked resolved; developers should verify that any failed calls during the windows were retried or handled gracefully. Source

OpenAI Codex Cloud Tasks performance degradation resolved

Codex Cloud Tasks (Web, CLI, App, VS Code extension) reported degraded performance earlier today. The issue is now resolved and services are fully operational. Teams using Codex should monitor for any lingering latency spikes. Source

OpenAI Responses API 404 spikes fixed after rollback

Between 4:05 pm and 4:40 pm PT on May 8, the Responses API returned elevated 404 errors due to a recent deployment. The deployment was rolled back and the service is back to normal. Review logs for any missed responses during that interval. Source
