Takeaway — Update anthropic to v0.101.0 for native AWS auth on Claude Platform; OpenAI resolved GPT-5.5 error spike across Chat Completions and Responses.

Anthropic's Python SDK v0.101.0 ships a new AWS client for Claude Platform on AWS. OpenAI's GPT-5.5 saw elevated errors across Chat Completions and the Responses API, fully resolved on May 11 at 17:59 UTC.

[INFO] Anthropic SDK v0.101.0 — native AWS client for Claude Platform on AWS

anthropic-sdk-python v0.101.0 adds a dedicated AWS client class for accessing Claude Platform on AWS, separate from the existing Amazon Bedrock integration. This lets teams on AWS infrastructure use IAM-based auth flows with Claude Platform directly, without routing through Bedrock. Upgrade with pip install -U anthropic. The release also patches a missing f-string prefix in a file type error message (cosmetic only, no API impact).

[INFO] OpenAI GPT-5.5 elevated errors resolved — Chat Completions and Responses API

An incident caused elevated error rates for gpt-5.5 across both Chat Completions and the Responses API; OpenAI marked full recovery at 17:59 UTC on May 11. No lasting impact is expected, but if your stack calls gpt-5.5 directly, check error logs for the window roughly May 11 15:00–18:00 UTC to confirm no requests were silently dropped. OpenAI's status page lists both services as operational.
