ApiDelta — May 8, 2026

Several LLM providers rolled out updates and resolved incidents, notably a new Mistral model and multiple Claude‑related error spikes.

Mistral Small 3.2 model and API go live

Mistral released the Small 3.2 model (model ID mistral-small-2506) and opened its dedicated endpoint, allowing developers to start sending requests to the new 250‑token‑optimized model. Existing integrations should update the model name in API calls to access the latest version. Source

Breaking change in Mistral Python SDK v2.4.5

The mistral.workflows.executions.stream() method now returns a different payload shape: response.[].data.data.union(CustomTaskInProgressResponse).attributes.payload replaces the previous structure. Code that parses streamed execution results will need to be updated to avoid runtime errors. Source

Claude platform incidents resolved

Claude experienced elevated error rates across multiple models (including Opus 4.7 and Sonnet 4.6) on May 6‑8. All incidents have been marked resolved and are now in monitoring. No code changes are required, but developers should be aware of the brief outage when reviewing recent logs. Source

Codex Cloud Tasks degraded performance

OpenAI reports a higher‑than‑expected failure rate for follow‑up prompts to Codex Cloud Tasks, causing degraded performance in the Codex web interface. Teams relying on Codex task chaining may see increased latency or retries; consider implementing exponential backoff until the issue is cleared. Source

GitHub IP‑restriction outage resolved

An incident that blocked connections for organizations restricting GitHub access by IP address was resolved on May 7. Services that depend on GitHub API calls from restricted IP ranges should now operate normally. Source
